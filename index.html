Introduction

PayByGroup works by passing information about a given purchase from your site to ours via a Javascript widget on your webpage. For example, if someone wants to start a PayByGroup from a vacation rental website, PayByGroup needs to know things like what property they want to rent, how much it will cost, and when they'll be staying there.

This page provides all the HTML and Javascript tools necessary to add this widget to your site. Start with the Javascript integration section to get the snippet of Javascript code to put in the footer of your site -- this snippet is necessary in order for any other PayByGroup widgets or informational units to work.

Next, use the Design assets section to find the pre-built widget that best fits the available space on the "checkout" page of your site. Remember that PayByGroup isn't trying to replace your existing checkout method, but rather is just a secondary option for users who aren't quite ready to complete the purchase on their own.

You can then customize the "Split the Cost" button inside those widgets with Data attributes. A good rule of thumb is that you should pass every data attribute that you CAN pass. For example, if you don't pass the attribute called cost, then we'll have to ask the customer to supply a total cost quote. If you don't pass name, the customer has to type in the name of the inventory item he's trying to buy.

If you have any questions about this page or how to use any of the code here, please reach out to us at support@paybygroup.com

Data attributes

In order to customize your PayByGroup you need to add the following data attributes to your pbg_info element.

Tip: Remember to add data attributes in the element in the following way data-name="Hello world" .

Group purchase

purchase-image-url:
string URL for the PayByGroup image
name:
string Name of the product/service for the PayByGroup
purchase-id:
string Your ID used to identify the purchase for your records
cost:
float Total cost of the purchase
currency:
string Three letter code of the currency being used (defaults to USD if not specified). No automatic conversions happen, so make sure that your gateway accepts it
link-url:
string URL for the page of the product/service
description:
string Description of the purchase


Other commonly used data attributes

These attributes are used on an as needed basis depending on your product/service.

start-date:
date YYYY-MM-DD If your service needs to store a start date. i.e. Reservation start date
end-date:
date YYYY-MM-DD If your service needs to store an end date. i.e. Reservation end date
deposit-amount:
float Cost of the deposit to hold the inventory
hold-deadline:
date YYYY-MM-DD Date on which the deposit expires and the inventory becomes subject to availability
purchase-inventory-id:
float Your ID used to identify the inventory. We use this to connect to your platform
min-min-people:
integer Absolute minimum number of spots that must be claimed in order for the purchase to complete
max-max-people:
integer Absolute maximum number of spots that are allowed to be claimed as part of the purchase 


<div class="pbg_info"
  data-purchase-image-url="https://example.com/picture.png"
  data-name="Awesome Escape"
  data-purchase-id="ESCAPE-23"
  data-cost="500.45"
  data-currency="CAD"
  data-link-url="http://example.com/property"
  data-description="Great rental for 3 people with lake view." >
</div>

Tip: This is where you should be sure to set the data attributes listed above and any additional ones you want to include .
